Bug 1: Coin extraction returns 0 because JSON.parse fails silently
=================================================================

Current state: status-server.cjs has an `extractCoins(logData)` function that
expects logData to be a parsed JSON object with `logData.stats.sessionCost`.

Reality: Bob CLI does NOT write JSON to logs/<task>.json. It writes plaintext
mixed with JSON-fragment substrings. The actual file contains lines like:

    "sessionCost": 0.027362
    "totalCalls": 1,
    "totalCoins": 0,

These are embedded in a much larger plaintext stream (PowerShell stderr, error
traces, etc.). JSON.parse on the whole file throws — and the current code
silently swallows it, returning 0.

Required fix: Replace `extractCoins()` to:
  1. Accept the raw file contents as a STRING (not parsed JSON).
  2. Use a regex to find ALL occurrences of `"sessionCost":\s*([\d.]+)` in
     the text. Each match represents one Bob session cost.
  3. Sum all matched values.
  4. Return the sum as a number.

The caller currently does something like:
    const logData = readJsonFile(logJsonFile);
    const coins = extractCoins(logData);

Update the caller to pass the raw text instead:
    const rawText = fs.readFileSync(logJsonFile, 'utf-8');
    const coins = extractCoins(rawText);

If the file does not exist, return 0 (no crash).

Verify with a quick test:
  - Read logs/v3-coin-fix.json — your regex should find at least one
    sessionCost value > 0 (the real run had ~0.027).
  - For tasks with no Bob log file (e.g. test-001), the function returns 0.

Mission: Create the PowerShell script that spawns one Bob in reviewer mode and produces .foreman/tasks/reviewer.json.

This script will be called by foreman-build.ps1 after all build workers have merged, AND can also be called standalone for the live demo.

Your scope (you may ONLY create/modify):
  - scripts/foreman-review.ps1 (NEW — your sole deliverable)

Files you must read first:
  - V3-PLAN.md at repo root (the spec — especially "Done-Definition" and the reviewer.json schema)
  - scripts/spawn-worker.ps1 (existing — see how a single Bob is spawned in v2)
  - scripts/foreman-build.ps1 (existing — see how Bob CLI is invoked from PowerShell)
  - .foreman-prompts/architecture.txt (any v2 prompt — see how prompts are loaded)

What you must NOT touch:
  - scripts/foreman-build.ps1 (another worker — v3-build-stage owns the integration)
  - scripts/spawn-worker.ps1 (frozen — read only, don't modify)
  - .foreman-prompts/reviewer.txt (another worker writes that — your script will load it)
  - status-server.cjs, dashboard/* (other workers)
  - V3-PLAN.md

Script contract:

```
.\scripts\foreman-review.ps1 [-Target <ref>] [-DryRun]
```

Parameters:
  - `-Target` (optional, default `master~6..HEAD`) — the git ref range to review
  - `-DryRun` (optional, default false) — if set, write the JSON but don't make commits

Behavior:
  1. Verify prerequisite files exist:
     - .foreman-prompts/reviewer.txt (must exist — fail fast with clear message if not)
     - .bob/rules-foreman-reviewer/01-reviewer.md (must exist)
     - The status-server should ideally be running, but the script must work without it (only writes JSON files)
  2. Create a worktree at worktrees/reviewer (clean if exists)
  3. Spawn a single Bob CLI process in worktrees/reviewer with:
     - Custom Mode: foreman-reviewer
     - Prompt content: .foreman-prompts/reviewer.txt + the resolved Target ref appended as context
     - Allow file writes within the worktree only
  4. Wait for the Bob process to exit (with a sensible timeout, e.g. 10 minutes — print warning if exceeded but do not kill)
  5. After Bob exits:
     - Verify .foreman/tasks/reviewer.json exists in the worktree
     - Copy it back to repo root .foreman/tasks/reviewer.json
     - Verify .foreman/reviewer-history.json exists in the worktree
     - Copy it back to repo root .foreman/reviewer-history.json
  6. Print one summary line: `Reviewer: N found, M fixed, K skipped (run_id ...)` parsed from the JSON
  7. Exit code 0 if reviewer.json was produced, 1 otherwise

Constraints:
  - Use only PowerShell built-ins and existing project conventions (the existing scripts use `git worktree add`, `Start-Process`, etc. — match that style)
  - DO NOT install PowerShell modules
  - DO NOT call the actual Bob CLI binary by hardcoded path — use the same invocation pattern as scripts/spawn-worker.ps1
  - The script must be idempotent — running twice in a row works (cleans worktree on second run)
  - All paths are relative to the repo root (script's parent's parent)

Verification before claiming done — ALL must pass:
  - Script exists and is syntactically valid PowerShell
  - Run with NO prerequisites: `.\scripts\foreman-review.ps1` exits 1 with clear error message about missing files
  - Place the prerequisite file `.foreman-prompts/reviewer.txt` (just any content for this test) and `.bob/rules-foreman-reviewer/01-reviewer.md` (already exists)
  - Run with `-DryRun`: `.\scripts\foreman-review.ps1 -DryRun -Target HEAD~1..HEAD`
    - It MUST run end-to-end without throwing (even if Bob session fails to actually do work — the script's own logic must not crash)
    - Cleans/creates worktrees/reviewer
    - Either succeeds with reviewer.json output or exits 1 with a clear message
  - Idempotent: running it twice in a row works (no leftover worktree blocking second run)
  - DONE means: -DryRun completes without script-level errors and produces meaningful output. Not "script parses". RUNS.
